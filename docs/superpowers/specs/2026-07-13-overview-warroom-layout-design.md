## 1. 問題

總覽頁從早期 2×2 儀表板，逐漸堆成「全寬 + 雙欄混排」長頁：

| 區塊 | 問題 |
|---|---|
| ASCII 總部 | 全寬 5–8 行，視覺權重大 |
| 今日 Token 收成 | 寬屏每 source 一列 ≈5 行，偏高 |
| 戰役狀態 · 宿敵路線 | 與經營 KPI 搶主屏；兩卡行數差大 |
| 公司 · 訓練/前沿 | 訓練卡常塞滿前沿算力明細，高度不齊 |
| 市佔 · 里程碑 | 相對平穩 |
| 董事會報告 · 產業動態 | 再兩張全寬，一屏難盡覽 |

`Grid` 只保證欄寬相等，**不保證同列等高**，畫面像補丁。玩家難以在 5 秒內抓住「公司好不好、token 收了多少」。

### 已確認方向

- **精簡 + 統一節奏**（資訊過載與視覺不齊都要解）
- 總覽主優先：**經營 KPI + 今日 Token 收成**
- 戰役細節：**整組搬離總覽**，新頁 **「戰情室」**
- 總覽骨架：**總部視覺錨 + 公司 KPI + 薄收成 + 下排狀態**
- Tab：**戰情室插在總覽後為 `2`**，其餘順延

## 2. 目標與非目標

### 目標

1. 總覽 5 秒內回答：估值/用戶/收成/訓練與算力壓力/里程碑。
2. 總覽 **0 張戰役內容卡**（戰略、宿敵、董事會、產業動態列表皆不在總覽）。
3. 同列卡片 **視覺高度一致**（pad，不改字級）。
4. 新增 **戰情室** 頁，承接全部戰役 UI 與 `e` 決策主場。
5. 純 TUI / layout；**不改** tick、經濟、戰役 sim。

### 非目標

- 改 sim / 經濟 / 戰役規則
- 新視覺語言（非卡片 HUD、圖表庫、時間軸）
- 戰情室可點選列表或地圖
- 總覽「顯示戰役」toggle
- 多日 token 歷史
- 自幹完整 responsive grid 引擎
- 刪除或合併成就頁

### 風格決策（已確認）

| 項 | 選擇 |
|---|---|
| 優化力度 | 方案 2：資訊分層 + 固定節奏 |
| 戰役去向 | 新頁「戰情室」 |
| 總覽骨架 | B：總部\|公司、全寬收成、三狀態卡 |
| Tab 位置 | `2` 戰情室，其後頁碼 +1 |
| 總覽戰役殘留 | 僅 footer 待決一行（有 Pending 時） |

## 3. 資訊架構與頁面職責

### 八頁導航

| 鍵 | 頁 | 原鍵 |
|---|---|---|
| 1 | 總覽 | 1 |
| 2 | **戰情室**（新） | — |
| 3 | 模型 | 2 |
| 4 | 市場 | 3 |
| 5 | 算力 | 4 |
| 6 | 團隊 | 5 |
| 7 | 科技 | 6 |
| 8 | 成就 | 7 |

- `numPages = 8`
- 頁常數重排：`PageOverview=0, PageWarRoom=1, PageModels=2, … PageAchievements=7`
- Tab 列、數字鍵 `1`–`8`、左右切頁皆跟 `pageNames`

### 職責邊界

| 頁 | 負責 | 不負責 |
|---|---|---|
| **總覽** | HQ 視覺、公司 KPI、今日 Token 收成、訓練/發佈摘要、消費者市佔摘要、里程碑&算力、待決戰役 **footer 提示** | 戰役狀態詳情、宿敵路線、董事會報告、產業動態列表 |
| **戰情室** | 公司戰略、宿敵路線、董事會報告、產業動態、`e` 決策主場 | 訓練開訓主 UI、模型定價、買算力主 UI |
| 其他頁 | 不變 | — |

### 資料流

- 僅搬移 / 重排 TUI renderer 與 layout kit。
- 總覽 footer 待決提示只讀 `state.Events.Pending`（長度 N > 0 時顯示）。
- 不新增 store 欄位、不改 meta schema。

### 成功標準

1. ≥100 欄時總覽結構為 **2 欄 + 全寬收成 + 3 等寬**，主資訊盡量一屏。
2. 同列卡片 `lipgloss.Height` 一致。
3. 戰役內容卡只出現在戰情室；總覽到戰情室 ≤ 1 鍵（`2`）。

## 4. 總覽版面規格

### 4.1 骨架（寬 ≥ 100）

```
┌─ Tokensmith ─ Day N ─ 資源條 ─ Tabs ─────────────┐
│ ┌─ 總部 (ASCII) ─────┐ ┌─ 公司 ────────────────┐ │
│ │  art / 階段        │ │ 估值 總用戶 排名 月收  │ │
│ │                    │ │ spark                 │ │
│ └────────────────────┘ └──────────────────────┘ │  ← EqualHeight
│ ┌─ 今日 Token 收成 · MM/DD ────────────────────┐ │
│ │ 四 source 橫排 totals + 合計（薄卡）          │ │
│ └──────────────────────────────────────────────┘ │
│ ┌訓練/發佈┐ ┌市佔(消費者)┐ ┌里程碑&算力┐        │
│ │ ≤5 行   │ │ ≤5 行      │ │ 3 行固定  │        │  ← 三等寬 + EqualHeight
│ └─────────┘ └───────────┘ └──────────┘        │
│ ⚠ 待決策 · [2]戰情室 [e] …   （無 Pending 則省略）│
│ footer                                         │
└─────────────────────────────────────────────────┘
```

### 4.2 各卡內容與行數

| 卡 | 內容 | Body 行數 | 裁切 / 規則 |
|---|---|---|---|
| **總部** | 現有 7 階段 ASCII；訓中微閃爍保留 | art 高度 | 與公司卡 EqualHeight；過高則少裝飾空行或沿用窄版單行階梯 |
| **公司** | 估值、總用戶、排名、月營收、spark 一行 | ≤5 | 無 spark 不留空行；不足 pad |
| **今日 Token 收成** | 四 source **total** + 合計 | 寬：1–2 行 source + 1 行合計；窄：compact ≤2 行 | **取消**每 source 一列的 5 行高卡；In/Out 細節不在總覽展開（Grok 估算可在 label 保留簡記） |
| **訓練/發佈** | 訓中：Gen + bar + 區隔；否則無訓/待發佈；前沿最多 **2 行**（進度% + ETA 或停滯） | ≤5 | **移除**前沿分配% / 有效→折合→建議等明細（科技頁已有） |
| **市佔** | 最多 4 名（含自己），你永遠顯示 | ≤5 | 超過截斷 |
| **里程碑&算力** | 訓練 util、推理 util、下一里程碑 | **3** | 固定三行 |

### 4.3 欄寬

| 列 | 佈局 |
|---|---|
| Row1 | 兩欄等寬 `(cw - gap) / 2` |
| Row2 | 全寬 |
| Row3 | 三等寬 `(cw - 2*gap) / 3` |

### 4.4 窄屏

| 寬度 | 行為 |
|---|---|
| `< 80` | 全部直向堆疊；總部單行階段條；不做 EqualHeight |
| `80–99` | Row1 左右兩欄；Row3 為 **上兩欄 + 第三卡全寬**（2+1）；收成 compact；同列做 EqualHeight |
| `≥ 100` | 完整骨架 + 三等寬 + 同列 EqualHeight |

### 4.5 總覽移除與保留

**移除（不再掛載於總覽）：**

- `renderCampaignStatusCard`
- `renderRivalRoadmapCard`
- `renderBoardReportCard`
- `renderEventsCard`

**保留：**

- `t` 等既有總覽快捷（行為不變）
- 經營向 `pressures()`「注意」卡（非戰役）：仍掛總覽最底，有警告才出現；與下方待決引導分開
- 待決引導 **一行**（非大卡，可併入 page 底或 notice 列）：  
  `⚠ 產業待決策 N · [2]戰情室  [e]決策`  
  僅當 `len(Pending) > 0`

## 5. 戰情室版面規格

### 5.1 定位

戰役專頁：讀戰情、看宿敵、處理產業決策。不放公司 KPI / Token 收成。

### 5.2 骨架

```
┌─ [2]戰情室 ─────────────────────────────────────┐
│ ┌─ 公司戰略 ──────────┐ ┌─ 宿敵路線 ──────────┐ │
│ │ doctrine / 階段     │ │ 主要宿敵 + 攪局者   │ │
│ │ 進度 / 決勝 / perk  │ │ 已確認 / 下一步     │ │
│ └─────────────────────┘ └─────────────────────┘ │  ← EqualHeight
│ ┌─ 產業動態 ──────────────────────────────────┐ │
│ │ 待決置頂 + 最近 log，最多 6 行                │ │
│ └─────────────────────────────────────────────┘ │
│ ┌─ 董事會報告 ────────────────────────────────┐ │
│ │ 最新週期 + 最多 6 條 entries                  │ │
│ └─────────────────────────────────────────────┘ │
│ footer: [e]決策 [c]能力 [P]結算(若可)  [1]總覽   │
└─────────────────────────────────────────────────┘
```

### 5.3 各卡

| 卡 | 來源 | 調整 |
|---|---|---|
| 公司戰略 | `renderCampaignStatusCard` | 邏輯保留；與宿敵 EqualHeight |
| 宿敵路線 | `renderRivalRoadmapCard` | 邏輯保留；過長 detail `Truncate` 至欄寬 |
| 產業動態 | `renderEventsCard` | 行數上限 **4 → 6**；待決置頂 |
| 董事會報告 | `renderBoardReportCard` | entries 上限 **4 → 6** |

排列理由：上排戰略 vs 宿敵對讀；產業動態在報告之上（決策優先於歷史）；長內容走 page viewport 捲動。

### 5.4 鍵位

| 鍵 | 行為 |
|---|---|
| `e` | 事件決策 dialog；**主場在戰情室**；全域/總覽仍可觸發（習慣不斷）但不顯示事件大卡 |
| `c` | 待選能力（既有，有 pending 時） |
| `P` | 路線勝利結算（既有，可結算時） |
| `2` / Tab | 進入本頁 |
| `1` | 回總覽（footer 可提示） |

不在 footer 廣告訓練/買卡等經營鍵，避免戰情室變第二總覽。全域鍵若仍生效，不特別說明。

### 5.5 空態

未開戰役 / 無情報 / 無事件 / 無報告：沿用現有 muted 文案，四卡仍在，不崩版。

### 5.6 窄屏

- `< 80`：四卡直向堆疊  
- `≥ 80`：上排雙欄，下兩張全寬

## 6. Layout kit

擴充 `internal/tui/layout.go`（或同層）：

| API | 職責 |
|---|---|
| `EqualHeight(parts ...string) []string` | 量測高度，較矮者底部補空白行至同高 |
| `HRowEqual(gap int, parts ...string) string` | EqualHeight 後 HRow |
| `GridN(cw, gap, cols int, cells ...func(w int) string)` | 支援 3 欄；奇數尾格全寬（與現 `Grid` 一致） |
| `CardLines(kind, w, title string, lines []string, maxBody int) string` | body 最多 maxBody 行，超出截斷 |

原則：

- 只 pad，不改字級 / bar 寬
- 窄屏直堆時不做 EqualHeight
- 截斷丟次要行（總覽前沿明細改為根本不渲染）

## 8. 測試策略

1. **總覽 render**：含「公司」「今日 Token」等；**不含**「公司戰略」「宿敵路線」「董事會報告」「產業動態」作為卡片標題（footer 待決文案除外）。
2. **戰情室 render**：四標題存在；未開戰役不 panic。
3. **等高**：左右行數不同時，`HRowEqual` 後高度相同。
4. **導航**：`2` → 戰情室；`8` → 成就；`numPages == 8`。
5. **迴歸**：`go test ./internal/tui/ ...`；`e`/`c` 從戰情室仍可開 dialog。

## 9. 錯誤與邊界

- 舊存檔、無 campaign、無 daily bucket：空態安全。
- 極窄終端：直堆、總部單行、收成 compact。
- `Pending` 空：總覽不顯示待決行。
- 頁碼 renumber：所有 help/footer/測試字串同步。
